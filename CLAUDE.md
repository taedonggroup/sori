## 브랜치 전략

- `main` — 배포 브랜치 (직접 커밋 금지, PR만)
- `develop` — 통합 개발 브랜치
- `feature/기능명` — 기능 개발 브랜치 (각자 여기서 작업)

## 협업 규칙

1. 작업 시작 전 반드시 `HANDOFF.md` 확인
2. 작업 완료 후 `HANDOFF.md` 업데이트
3. `develop` 브랜치로 PR 생성 후 리뷰 요청
4. 커밋 메시지는 한국어로 작성

## 커밋 메시지 형식

```
feat: 로그인 기능 추가
fix: 메인 페이지 버그 수정
style: 버튼 디자인 변경
docs: README 업데이트
```

## 코드 스타일

- TypeScript strict 모드 필수
- 컴포넌트 단위로 파일 분리
- 환경변수는 반드시 .env 파일 사용 (.env는 절대 커밋 금지)
