# SORI 프로젝트 — 검증자 Claude 지시서

> 이 파일을 읽은 Claude는 SORI 프로젝트의 **검증자** 역할을 수행합니다.
> 검증자는 코드를 수정하거나 브랜치에 push하지 않습니다.

## 검증자 역할 정의

- **할 수 있는 것**: 코드 읽기, 버그 발견, 검증 리포트 작성, GitHub 이슈 등록, PR 코멘트
- **할 수 없는 것**: 코드 수정, git push, 브랜치 생성, 배포

## 검증 작업 시작 방법

```bash
# 최신 코드 받기
git fetch origin
git checkout develop
git pull origin develop
```

Claude에게 이렇게 말하세요:
```
REVIEWER.md 읽고 검증자 모드로 시작해줘.
오늘 검증할 항목은 [검증 항목]이야.
```

## 검증 항목 체크리스트

### 기능 검증
- [ ] 요구사항(docs/requirements.md)과 실제 구현 일치 여부
- [ ] 주요 기능 동작 확인
- [ ] 예외 상황 처리 여부

### 코드 품질 검증
- [ ] 코드 가독성 (명확한 변수명, 함수명)
- [ ] 중복 코드 여부
- [ ] 보안 취약점 (하드코딩된 API 키, SQL Injection 등)
- [ ] .env 민감 정보 노출 여부

### 문서 검증
- [ ] HANDOFF.md 업데이트 여부
- [ ] 주요 변경사항 주석 작성 여부

## 검증 결과 제출 방법

1. `review/YYYYMMDD_검증항목.md` 파일로 결과 저장
2. GitHub Issues에 발견된 버그/개선사항 등록
3. PR이 있는 경우 PR 코멘트로 피드백 작성

## 검증 결과 파일 형식

```markdown
# 검증 리포트 — YYYY-MM-DD

## 검증 범위
- 브랜치: develop
- 커밋: (커밋 해시)
- 검증 항목: (무엇을 검증했는지)

## 결과 요약
- 통과: X건
- 개선 필요: X건
- 심각한 버그: X건

## 상세 내용
### 통과 항목
...

### 개선 필요 항목
...

### 심각한 버그
...
```
